A type-safe builder over Convex's native `searchIndex` — index names, search fields, and `eq` filters are checked against your schema.

## Fluent `search(...)`

```ts
import { search } from 'nuxt-backend/convex/search'

const results = await search(ctx, 'messages')
  .withSearchIndex('search_text')
  .search('text', term)
  .eq('userId', userId)
  .take(20)
```

## `defineSearch(...)`

`defineSearch(query, { table, index, searchField, defaultLimit? })` turns that into a ready-made query taking `{ query, limit? }`, which the `useSearch` composable drives reactively.

```ts [backend/search.ts]
import { defineSearch } from 'nuxt-backend/convex/search'
import { query } from './_generated/server'

export const searchMessages = defineSearch(query, {
  table: 'messages',
  index: 'search_text',
  searchField: 'text',
  defaultLimit: 20,
})
```

## Reactive `useSearch()`

Debounced, reactive full-text search. Pauses while the term is blank.

```vue
<script setup lang="ts">
import { api } from '#backend/api'

const term = ref('')
const { results, isLoading } = useSearch(api.search.searchMessages, term, { debounce: 200 })
</script>

<template>
  <input v-model="term" placeholder="Search…">
  <ul>
    <li v-for="r in results" :key="r._id">{{ r.text }}</li>
  </ul>
</template>
```
