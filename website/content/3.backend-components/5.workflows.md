`setupWorkflows(components.workflow, options?)` returns a `WorkflowManager` with sensible default retry behavior. Define durable functions with `workflow.define({ args, handler })` and start them with `workflow.start(ctx, internal.x.y, args)`.

```ts [backend/workflows.ts]
import { setupWorkflows } from 'nuxt-backend/convex/workflows'
import { components, internal } from './_generated/api'

export const workflow = setupWorkflows(components.workflow)

export const onSignup = workflow.define({
  args: { userId: v.string(), email: v.string(), name: v.optional(v.string()) },
  handler: async (step, args) => {
    // durable steps — each is retried independently and survives restarts
    await step.runMutation(components.backend.email.send, {
      to: args.email,
      subject: 'Welcome!',
      html: `<p>Hi ${args.name ?? 'there'} 👋</p>`,
    })
  },
})
```

## Welcome-email flow

The scaffold kicks off a durable welcome-email workflow on signup via the `onUserCreated` integration hook, which sends through `components.backend.email.send`:

```ts [backend/auth.ts]
export const { authComponent, createAuth, getAuthUser } = setupAuth(components.backend, query, {
  integrations: {
    rateLimiter,
    onUserCreated: async (ctx, user) => {
      await workflow.start(ctx, internal.workflows.onSignup, {
        userId: user.id, email: user.email, name: user.name,
      })
    },
  },
})
```

::tip
The `ctx` passed to `onUserCreated` is a full mutation/action ctx, so it can start a Workflow directly. See [cross-component wiring](/convex-backend/auth-setup#cross-component-wiring).
::

## Reactive status

Track a running workflow from the client — `useWorkflowStatus` pauses while the id is `null`/`undefined`:

```vue
<script setup lang="ts">
import { api } from '#backend/api'

const status = useWorkflowStatus(api.workflows.status, workflowId)
</script>
```
