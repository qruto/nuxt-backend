title: Aggregates
description: Denormalized counts and sums kept in sync automatically, read reactively with useCount.

Construct one `TableAggregate` per mounted instance and keep it in sync automatically with triggers.

```ts [backend/aggregates.ts]
import { TableAggregate, Triggers, withTriggers } from 'nuxt-backend/convex/aggregate'
import { components } from './_generated/api'
import { mutation as rawMutation } from './_generated/server'
import type { DataModel } from './_generated/dataModel'

export const messagesCount = new TableAggregate<{
  Key: null
  DataModel: DataModel
  TableName: 'messages'
}>(components.aggregate, { sortKey: () => null })

const triggers = new Triggers<DataModel>()
triggers.register('messages', messagesCount.trigger())

// Use this wrapped `mutation` instead of the raw one so writes keep the aggregate in sync.
export const mutation = withTriggers(rawMutation, triggers)
```

Expose a query that reads the aggregate, then read it reactively on the client:

```vue
<script setup lang="ts">
import { api } from '#backend/api'

// Live aggregate count (coerces loading/null to 0). `useCount` is an alias.
const messageCount = useCount(api.aggregates.countMessages)
</script>
```

::note
`useAggregate` and `useCount` are the same composable; `useCount` reads better when you're counting rows.
::
