The [Resend](https://www.convex.dev/components/resend) component is **nested inside `backend`**, so email works out of the box with no app-level mount and no setup helper — just set `RESEND_API_KEY`.

## Sending mail

Call it from any app function (action, mutation, or a Workflow step):

```ts
await ctx.runMutation(components.backend.email.send, {
  to: 'user@example.com',
  subject: 'Welcome!',
  html: '<p>Hello</p>',
})
```

It enqueues a **durable** email and resolves to the email id, or logs and resolves to `null` when `RESEND_API_KEY` is unset.

## Auth emails are automatic

The same path powers auth OTP / verification / reset automatically — see [cross-component wiring](/convex-backend/auth-setup#cross-component-wiring). You don't pass an `email` integration.

## Test mode

| Variable | Effect |
|---|---|
| `RESEND_API_KEY` | When unset, mail logs to the console instead of sending |
| `RESEND_FROM` | Sender address; falls back to Resend's shared onboarding sender |
| `RESEND_TEST_MODE` | Defaults **on** — only Resend's test addresses receive mail. Set to `"false"` to deliver to real recipients |

::warning
With `RESEND_TEST_MODE` left on (the default), only Resend's test addresses receive mail. Set it to `"false"` once you've verified a sending domain to deliver to real recipients.
::
