title: Rate Limiting
description: Throttle auth flows and your own endpoints with the rate-limiter component, pre-seeded with sensible auth defaults.

`setupRateLimiter(components.rateLimiter, limits?)` returns a `RateLimiter` pre-seeded with `DEFAULT_AUTH_LIMITS` (`emailOtp`, `signIn`, `signUp`, `passwordReset`). Your `limits` are merged on top.

```ts [backend/rateLimiter.ts]
import { setupRateLimiter } from 'nuxt-backend/convex/rate-limit'
import { components } from './_generated/api'

export const rateLimiter = setupRateLimiter(components.rateLimiter, {
  // your own limits, merged over DEFAULT_AUTH_LIMITS
  sendMessage: { kind: 'token bucket', rate: 10, period: 60_000 },
})
```

Pass the limiter into `setupAuth` to throttle OTP sends — see [cross-component wiring](/convex-backend/auth-setup#cross-component-wiring).

::note
`DEFAULT_AUTH_LIMITS` is also exported from `nuxt-backend/convex/rate-limit` if you want to inspect or extend the defaults directly.
::
