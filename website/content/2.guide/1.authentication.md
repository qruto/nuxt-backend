Better Auth is wired in by the module. App code talks to it through a single composable — `useAuth()` — plus an opt-in route middleware and three auth-state components.

## `useAuth()`

`useAuth()` is the single Better Auth service. It exposes the Better Auth client, the reactive session wrapper, and the computed auth state used by the Convex integration:

```vue
<script setup lang="ts">
const { client, session, isAuthenticated, isLoading } = useAuth()

await client.signIn.email({ email: 'user@example.com', password: 'secret' })
const user = computed(() => session.value.data?.user)
</script>
```

Use the same `client` for sign-up and sign-out flows.

## Signing in

```vue [pages/login.vue]
<script setup lang="ts">
const { client: auth } = useAuth()
const email = ref('')
const password = ref('')
const pending = ref(false)
const error = ref<string | null>(null)

async function signIn() {
  pending.value = true
  error.value = null
  try {
    await auth.signIn.email({ email: email.value, password: password.value })
    await navigateTo('/account')
  }
  catch (cause) {
    error.value = cause instanceof Error ? cause.message : 'Unable to sign in'
  }
  finally {
    pending.value = false
  }
}
</script>

<template>
  <form @submit.prevent="signIn">
    <input v-model="email" type="email" placeholder="Email">
    <input v-model="password" type="password" placeholder="Password">
    <button :disabled="pending" type="submit">Sign in</button>
    <p v-if="error">{{ error }}</p>
  </form>
</template>
```

## Reading session state

```vue [pages/account.vue]
<script setup lang="ts">
import { api } from '#backend/api'

definePageMeta({ middleware: 'auth' })

const { session } = useAuth()
const currentUser = useQuery(api.auth.getAuthUser, {})
const email = computed(() => session.value.data?.user.email)
</script>

<template>
  <section>
    <p>Signed in as {{ email }}</p>
    <pre>{{ currentUser }}</pre>
  </section>
</template>
```

`useAuth().session` exposes the Better Auth session, and `useQuery(...)` keeps Convex data live after hydration.

## Protecting routes

Guard a page with the built-in `auth` middleware — it's registered **non-global**, so installing the module never guards routes by surprise:

```vue
<script setup lang="ts">
definePageMeta({ middleware: 'auth' })
</script>
```

Unauthenticated users are redirected to `/login`.

## Auth-state components

Three components render their slot based on auth state — useful for headers, menus, and loading skeletons:

```vue
<template>
  <Authenticated>Signed in — show the dashboard</Authenticated>
  <Unauthenticated>Show a sign-in button</Unauthenticated>
  <AuthLoading>Resolving auth…</AuthLoading>
</template>
```

| Component | Renders its slot when |
|---|---|
| `<Authenticated>` | the user is authenticated |
| `<Unauthenticated>` | the user is unauthenticated |
| `<AuthLoading>` | auth state is still resolving |

## Advanced: low-level auth helpers

The package also exports low-level Convex auth helpers for custom auth integrations: `provideConvexAuth`, `useConvexAuth`, and the components above. These are building blocks for custom auth providers and are **not required** for the default Better Auth setup scaffolded by the module.

```ts
function useConvexAuth(): { isLoading: boolean, isAuthenticated: boolean }
function provideConvexAuth(options: ConvexAuthProviderOptions): ConvexAuthState
```
