The module auto-imports a set of server utilities in the `server/` directory and Nitro handlers. They mirror the official `convex/nextjs` API surface. There is no server-side injected Convex client — all server access goes through these helpers.

## Fetching data on the server

```ts [server/api/example.ts]
export default defineEventHandler(async () => {
  const messages = await fetchQuery(api.messages.list, {})
  await fetchMutation(api.messages.send, { body: 'From server' })
  const result = await fetchAction(api.images.generate, { prompt: 'cat' })
  return messages
})
```

| Function | Description |
|---|---|
| `fetchQuery` | Run a query (optionally pass `{ token }` for auth) |
| `fetchMutation` | Run a mutation (optionally pass `{ token }` for auth) |
| `fetchAction` | Run an action (optionally pass `{ token }` for auth) |
| `preloadQuery` | Preload query data for SSR |
| `preloadedQueryResult` | Extract the result from a preloaded payload |
| `backendAuth` | Authenticated server helpers for the current H3 event |

## Authenticated calls — `backendAuth(event)`

`backendAuth(event)` is the Nuxt/Nitro equivalent of the official Next helper. It can fetch the Convex auth token, check authentication, run authenticated queries/mutations/actions, and preload authenticated data.

```ts [server/api/profile.ts]
import { api } from '#backend/api'

export default defineEventHandler(async (event) => {
  const auth = backendAuth(event)

  if (!await auth.isAuthenticated()) {
    throw createError({ statusCode: 401, statusMessage: 'Unauthorized' })
  }

  return auth.fetchAuthQuery(api.users.me, {})
})
```

`backendAuth(event)` returns these helpers:

| Helper | Description |
|---|---|
| `handler` | Proxy Better Auth requests from Nuxt to the Convex site URL |
| `getToken` | Fetch a Convex auth token from the Better Auth session |
| `isAuthenticated` | Whether the current request has an authenticated Convex token |
| `fetchAuthQuery` | Run an authenticated Convex query |
| `fetchAuthMutation` | Run an authenticated Convex mutation |
| `fetchAuthAction` | Run an authenticated Convex action |
| `preloadAuthQuery` | Preload an authenticated query for `usePreloadedAuthQuery` |

::tip
Use `auth.getToken()` if you need to pass the token to a lower-level `fetchQuery`/`fetchMutation`/`fetchAction` call yourself.
::

## Preloading for SSR

`preloadQuery` returns a `Preloaded<Query>` payload designed to be passed back to app code and consumed with `usePreloadedQuery` — the server result is shown immediately, then the client goes live.

```ts [server/api/messages.preload.ts]
export default defineEventHandler(async () => {
  return preloadQuery(api.messages.list, {})
})
```

```vue
<script setup lang="ts">
const { data: preloaded } = await useFetch('/api/messages.preload')
const messages = usePreloadedQuery(preloaded.value!)
</script>
```

### Authenticated preloading

For auth-protected data, preload through `backendAuth(event)` and hydrate with `usePreloadedAuthQuery`:

```ts [server/api/account.preload.ts]
import { api } from '#backend/api'

export default defineEventHandler((event) => {
  return backendAuth(event).preloadAuthQuery(api.auth.getAuthUser, {})
})
```

```vue
<script setup lang="ts">
const { data: preloaded } = await useFetch('/api/account.preload')
const currentUser = usePreloadedAuthQuery(preloaded.value!)
</script>
```

`usePreloadedAuthQuery` keeps the server result visible while Convex auth is loading in the browser, skips the live query if the user is unauthenticated, and switches to the live authenticated query once client auth is ready.
