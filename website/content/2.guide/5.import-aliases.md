The module registers `#backend` import aliases for both **Vite** and **Nitro**, pointing at your Convex functions directory (`backend/` or `convex/`) and its [generated files](https://docs.convex.dev/generated-api/).

```ts
import { api, internal, components } from '#backend/api'
import { query, mutation, action } from '#backend/server'
import type { Doc, Id } from '#backend/dataModel'
```

| Alias | Resolves to | Exports |
|---|---|---|
| `#backend` | your Convex functions dir (`backend/` or `convex/`) | — |
| `#backend/api` | `_generated/api` | `api`, `internal`, `components` |
| `#backend/server` | `_generated/server` | `query`, `mutation`, `action`, `*Ctx`, … |
| `#backend/dataModel` | `_generated/dataModel` | `DataModel`, `Doc`, `Id`, `TableNames` |
| `#backend/_generated` | `_generated` (long form) | any generated file |

The short aliases resolve to the same files as `#backend/_generated/*`, so use whichever you prefer.

::warning
Because `#backend/api`, `#backend/server`, and `#backend/dataModel` map straight to the generated modules, they **shadow** any function file you happen to name `api.ts` / `server.ts` / `dataModel.ts`. Import those via a different name (or `#backend/_generated/...`).
::
