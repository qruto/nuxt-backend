```ts
import nuxtBackendModule, { type ModuleOptions } from 'nuxt-backend'

interface ModuleOptions {
  url?: string
  siteUrl?: string
  authRoute?: string                  // default '/api/auth'
  installation?: 'default' | 'local'  // default 'default'
}
```

The module also augments runtime config:

- `RuntimeConfig.backend.siteUrl`
- `PublicRuntimeConfig.backend.{ url, siteUrl }`

## Provided vs. intentionally app-owned

`nuxt-backend` deliberately uses only the part of the Nuxt module API that an auth/data integration needs, and leaves routing, layout, and styling to your app.

| Capability (Nuxt Kit / mechanism) | In `nuxt-backend` |
|---|---|
| Module options · `defineNuxtModule({ configKey })` | ✅ `backend.url`, `siteUrl`, `authRoute`, `installation` |
| Runtime config · `options.runtimeConfig` | ✅ public + private `backend` keys |
| App plugins · `addPlugin` | ✅ client + server plugins, injects `$convex` |
| Auto-imported composables · `addImports` | ✅ Convex + auth composables |
| Auto-imported components · `addComponent` | ✅ `<Authenticated>`, `<Unauthenticated>`, `<AuthLoading>` |
| Route middleware · `addRouteMiddleware` | ✅ `auth` (opt-in, non-global) |
| Server routes · `addServerHandler` | ✅ auth proxy at `authRoute/**` |
| Server auto-imports · `addServerImports` | ✅ `fetchQuery`, `preloadQuery`, `backendAuth`, … |
| Import aliases · `options.alias` (+ `nitro.alias`) | ✅ `#backend`, `#backend/api`, `#backend/server`, `#backend/dataModel`, `#backend/_generated` |
| Type augmentation · `declare module` | ✅ runtime config + generated declarations |
| Package entrypoints · `exports` map | ✅ module, Convex auth bridge, per-component helpers |
| Scaffolded files on disk | ✅ first-run Convex wiring |
| Layouts · `addLayout` | — app-owned |
| Pages / routes · `extendPages` | — app-owned |
| Route rules · `extendRouteRules` | — app-owned |
| Global head / CSS · `app.head` / `options.css` | — app-owned |
| Global route middleware · `addRouteMiddleware({ global: true })` | — app-owned (auth is opt-in) |
| Nitro plugins · `addServerPlugin` | — not used |
| Bundler plugins · `addVitePlugin` / `extendViteConfig` | — not used |
| Module dependencies · `installModule` | — not used |
| Nuxt layer · `extends` | — not shipped as a layer |

::note
Keeping pages, layouts, global CSS, global middleware, route rules, and layers app-owned means installing the module never changes your routing, layout, styling, or domain behavior.
::
