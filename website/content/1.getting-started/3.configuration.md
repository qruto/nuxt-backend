All options have sensible defaults, and environment variables are picked up automatically. The minimal config is just the module:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-backend'],
})
```

## Module options

The `backend` options are optional. The module first reads `url` and `siteUrl` from explicit module options, then falls back to the canonical Nuxt app environment variables.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-backend'],
  backend: {
    url: 'https://your-deployment.convex.cloud',
    siteUrl: 'https://your-deployment.convex.site',
    authRoute: '/api/auth',
  },
})
```

::field-group
  :::field{name="url" type="string"}
  Convex deployment URL. Falls back to `NUXT_PUBLIC_CONVEX_URL`.
  :::

  :::field{name="siteUrl" type="string"}
  Convex site URL used by the auth proxy. Falls back to `NUXT_PUBLIC_CONVEX_SITE_URL`.
  :::

  :::field{name="authRoute" type="string" defaultValue="'/api/auth'"}
  Path of the same-origin Better Auth proxy. If you change it, keep `createAuthOptions` and the Convex auth config on the same path.
  :::

  :::field{name="installation" type="'default' | 'local'" defaultValue="'default'"}
  Whether the first scaffold uses the packaged component defaults or creates a local hybrid component for schema ownership. See [Local installation](/convex-backend/local-installation).
  :::
::

Auth is always enabled by the module. The only installation choice is whether the first scaffold uses the packaged component defaults or a local hybrid component.

## Runtime config

The module resolves these from module options or env vars and exposes them as typed runtime config:

- `runtimeConfig.public.backend.url`
- `runtimeConfig.public.backend.siteUrl`
- `runtimeConfig.backend.siteUrl`

## Environment variables

### App variables

Minimal app environment file:

```bash [.env.local]
NUXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
NUXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment.convex.site
```

### Convex deployment variables

Required Convex dashboard variables:

| Variable | Where | Description |
|---|---|---|
| `NUXT_PUBLIC_CONVEX_URL` | `.env.local` / `.env` | Convex deployment URL used by the Nuxt app |
| `NUXT_PUBLIC_CONVEX_SITE_URL` | `.env.local` / `.env` | Convex site URL used by the Nuxt auth proxy |
| `SITE_URL` | Convex dashboard | App URL such as `https://nuxt-backend.localhost` |
| `BETTER_AUTH_SECRET` | Convex dashboard | Secret used by Better Auth |

### Component environment variables

The bundled components add the following **optional** variables. The scaffolded `convex.config.ts` **declares** them on the deployment via Convex's typed [`defineApp({ env })`](https://docs.convex.dev/production/environment-variables), so they're validated at deploy and read type-safely through `env` from `_generated/server`. Leave any unset and the matching feature stays a graceful no-op.

| Variable | Feature | Effect when unset |
|---|---|---|
| `RESEND_API_KEY` | Email (Resend) | OTP / emails log to the console instead of sending |
| `RESEND_FROM` | Email | Falls back to Resend's shared onboarding sender |
| `RESEND_TEST_MODE` | Email | Defaults to test mode (only delivers to Resend's test addresses); set to `"false"` to send to real recipients |
| `POLAR_ORGANIZATION_TOKEN` | Billing (Polar) | Checkout/portal disabled; subscription queries return `null` |
| `POLAR_WEBHOOK_SECRET` | Billing | Polar webhook events aren't verified |
| `POLAR_SERVER` | Billing | Defaults to `sandbox` |

```bash
npx convex env set RESEND_API_KEY re_...
npx convex env set POLAR_ORGANIZATION_TOKEN polar_...
```

::note
The `RESEND_*` vars are forwarded to the nested `backend` component (`app.use(backend, { env })`), since [components can't read the app's env directly](https://docs.convex.dev/components). The scaffolded `convex.config.ts` handles this for you — see [Backend components overview](/backend-components/overview).
::

## Changing the auth route

If you change `backend.authRoute` from `/api/auth`, keep the same path everywhere — the Nuxt proxy route, Better Auth `basePath`, and `defineBackendAuthConfig({ basePath: ... })` must all match. See [Customizing auth](/convex-backend/customizing-auth).
