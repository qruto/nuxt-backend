The package ships two things in one:

| Layer | What it does |
|---|---|
| **Nuxt module** (`nuxt-backend`) | Registers plugins, composables, server utilities, the auth proxy, and auto-scaffolds Convex root files |
| **Convex component** (`nuxt-backend/convex/component/convex.config`) | Defines the `backend` component with a Better Auth adapter proxy and auth config |

## Request flows

Three transports run side by side, each with its own path through the stack:

::card-group
  :::card{icon="i-lucide-zap" title="Real-time"}
  `Browser ⇄ Convex Cloud` over a **WebSocket** (`ConvexVueClient`) — queries subscribe, Convex pushes live updates.
  :::

  :::card{icon="i-lucide-server" title="SSR"}
  `Browser → Nuxt → Convex` over HTTP (`ConvexHttpClient`) — the server renders HTML and ships a hydration payload.
  :::

  :::card{icon="i-lucide-shield-check" title="Auth"}
  `Browser → /api/auth/* → Nuxt proxy → Better Auth → Convex` — the session token and `Set-Cookie` return same-origin.
  :::
::

### Real-time data

The client `ConvexVueClient` connects over a **WebSocket** for real-time reactivity. `useQuery`, `useMutation`, `useAction`, and pagination subscribe through this client and update live.

::demo-connection
::

### SSR

On the server, `fetchQuery`, `fetchMutation`, `fetchAction`, and `preloadQuery` use a `ConvexHttpClient` under the hood. Pass `{ token }` when authentication is required. `preloadQuery` produces a payload that hydrates seamlessly into the live client via `usePreloadedQuery`.

### Authentication

`/api/auth/*` is **proxied** from Nuxt to the Convex site URL so cookies stay same-origin. Better Auth persists users and sessions through the Convex adapter inside the `backend` component, and issues a session token the client uses to authenticate its WebSocket.

::note
This mirrors the official Convex + Better Auth integration: real-time over WebSocket, SSR over HTTP, and a same-origin auth proxy keeping cookies on your domain.
::
