`nuxt-backend` integrates [Convex](https://convex.dev) with [Nuxt](https://nuxt.com) — one package that ships a **Nuxt module** and a **Convex auth component** with [Better Auth](https://www.better-auth.com) built in.

It is a Vue/Nuxt port of the official Convex + Better Auth React/Next integration. It keeps the same core pieces — the Better Auth Convex plugin, Convex auth config, server helpers, auth-aware preloading, and client auth state — but preconfigures the route wiring and component setup so new Nuxt apps do less manual installation work.

::note
This site covers the full surface. For a one-command start, jump straight to [Installation](/getting-started/installation).
::

## Two layers in one package

The package ships two things that work together:

::card-group
  :::card{icon="i-simple-icons-nuxtdotjs" title="Nuxt module"}
  Registers plugins, auto-imported composables and components, server utilities, the same-origin auth proxy, import aliases, and auto-scaffolds the Convex root files. Installed with `modules: ['nuxt-backend']`.
  :::

  :::card{icon="i-lucide-database" title="Convex auth component"}
  Defines the `backend` Convex component — Better Auth email/password out of the box, a Convex adapter for persistence, and HTTP auth endpoints served through Convex. Mounted in your `convex.config.ts`.
  :::
::

The intended integration shape:

- **Nuxt** owns the UI, route middleware, SSR handlers, and the same-origin auth proxy.
- **Convex** owns data, real-time subscriptions, Better Auth persistence, and auth identity.
- **`nuxt-backend`** keeps the auth route, auth config, and client/server helpers aligned.

## What the module provides

Adding `modules: ['nuxt-backend']` registers the following surface:

- **Configuration** — a `backend` config key (`url`, `siteUrl`, `authRoute`, `installation`) plus public/private runtime config, all falling back to env vars.
- **App runtime (Vue)** — a client Convex plugin and an SSR auth-token prefetch plugin (injects `$convex`); auto-imported composables (`useQuery`, `useMutation`, `useAuth`, …); auto-registered components (`<Authenticated>`, `<Unauthenticated>`, `<AuthLoading>`); and an opt-in `auth` route middleware.
- **Server (Nitro)** — a same-origin Better Auth proxy at `authRoute/**`, plus server auto-imports (`fetchQuery`, `preloadQuery`, `backendAuth`, …).
- **Project wiring & types** — `#backend` import aliases for both Vite and Nitro, first-run scaffolded Convex files, and full type augmentation.
- **Convex auth bridge** — package entrypoints you import inside your Convex functions (`nuxt-backend/convex`, `nuxt-backend/convex/app`, the per-component setup helpers, …).
- **Backend components** — bundled [Convex components](https://www.convex.dev/components) (billing, email, rate limiting, workflows, migrations, aggregates, search) wired into Better Auth and activated by env vars.

::tip
The module deliberately uses only the part of the Nuxt module API that an auth/data integration needs. Pages, layouts, global CSS, global middleware, and route rules stay **app-owned** — installing the module never changes your routing, layout, styling, or domain behavior.
::

## Official integration parity

`nuxt-backend` ports the official Better Auth Convex React/Next integration to Vue/Nuxt, giving each moving part Nuxt defaults:

| Official React/Next piece | Nuxt/Vue equivalent |
|---|---|
| Manual Better Auth Convex component files | Packaged `nuxt-backend/convex/component/convex.config` by default, or local `backend/components/backend/*` with `installation: 'local'` |
| `auth.config.ts` using the Better Auth Convex provider | `nuxt-backend/convex/auth.config`, scaffolded as `backend/auth.config.ts` |
| `auth.ts` with `authComponent`, `createAuthOptions`, `options`, `createAuth` | Scaffolded `backend/auth.ts` using `setupAuth(...)` |
| Better Auth client with `convexClient()` | Auto-created Vue Better Auth client exposed through `useAuth()` |
| `ConvexBetterAuthProvider` | Nuxt client plugin plus `useConvexAuth()` / `provideConvexAuth()` state |
| Next route handler proxy | Nuxt server proxy at `backend.authRoute` (default `/api/auth`) |
| `convexBetterAuthNextJs()` | `backendAuth(event)` for H3/Nitro handlers |
| `usePreloadedAuthQuery()` | `usePreloadedAuthQuery()` for Vue hydration |

The goal is API parity where it matters, with less setup: Nuxt owns route registration, app scaffolding, composable auto-imports, and the same-origin auth proxy.
