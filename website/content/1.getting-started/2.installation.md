## Prerequisites

- A [Convex](https://convex.dev) deployment (`npx convex dev` creates one).
- Node and a package manager. The bundled Convex components ship as dependencies of `nuxt-backend`, so this is the **only package you install**.

::steps{level="3"}

### Install the package

::code-group
```bash [nuxi]
npx nuxi@latest module add nuxt-backend
```

```bash [pnpm]
pnpm add nuxt-backend
```

```bash [npm]
npm install nuxt-backend
```

```bash [yarn]
yarn add nuxt-backend
```
::

::note{to="#using-strict-pnpm"}
The bundled components (`@convex-dev/aggregate`, `migrations`, `polar`, `rate-limiter`, `resend`, `workflow`, `better-auth`) come in transitively ‚Äî you never add them to your own `package.json`. Strict pnpm needs one extra line; see the note below.
::

### Add the Nuxt module

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-backend'],
})
```

### Configure environment variables

Create a `.env.local` in the app root. These are the only app variables auto-detected by the module:

```bash [.env.local]
NUXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
NUXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment.convex.site
```

Set the Convex **deployment** variables via the CLI (or the [dashboard](https://dashboard.convex.dev)):

```bash
npx convex env set BETTER_AUTH_SECRET=$(openssl rand -base64 32)
npx convex env set SITE_URL https://nuxt-backend.localhost
```

So the required values split like this:

| Location | Variables |
|---|---|
| `.env.local` (Nuxt app) | `NUXT_PUBLIC_CONVEX_URL`, `NUXT_PUBLIC_CONVEX_SITE_URL` |
| Convex dashboard | `SITE_URL`, `BETTER_AUTH_SECRET` |

::tip{to="/getting-started/configuration#component-environment-variables"}
The bundled components add a handful of **optional** Convex variables (`RESEND_API_KEY`, `POLAR_ORGANIZATION_TOKEN`, ‚Ä¶). Leave any unset and that feature stays a graceful no-op. See the configuration reference.
::

### Start the app once to scaffold the backend

```bash
npm run dev
```

On first run the module creates the minimum Convex files needed to mount the packaged Better Auth component and register the auth routes. If a Convex functions directory already exists it is reused; otherwise `backend/` is created:

```text
your-project/
‚îú‚îÄ‚îÄ backend/
‚îÇ   ‚îú‚îÄ‚îÄ convex.config.ts
‚îÇ   ‚îú‚îÄ‚îÄ auth.config.ts

‚îÇ   ‚îî‚îÄ‚îÄ http.ts
‚îî‚îÄ‚îÄ convex.json
```

`convex.json` is only generated when the functions directory is non-standard (anything other than `convex/`).

### Start Convex

```bash
npx convex dev
```

Deploy with `npx convex deploy`.

::

## Optional: override module defaults

Most apps do not need a `backend` block. Add it only to override the env-based defaults or change the auth route:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-backend'],
  backend: {
    url: 'https://your-deployment.convex.cloud',
    siteUrl: 'https://your-deployment.convex.site',
    authRoute: '/api/auth',
  },
})
```

## Using strict pnpm

A flat `node_modules` (npm, yarn, or pnpm with `nodeLinker: hoisted`) exposes the bundled components automatically. pnpm's default isolated linker hides transitive deps, so let Convex's components reach the project root by adding to `.npmrc`:

```ini [.npmrc]
public-hoist-pattern[]=@convex-dev/*
```

(or set `node-linker=hoisted`). Without this, the `convex dev` push fails to resolve the component definitions.

::warning
The scaffolded `convex.config.ts` imports the bundled components by their **real specifiers** (e.g. `@convex-dev/polar/convex.config`). Convex builds its component tree from those `‚Ä¶/convex.config` imports ‚Äî they can't be re-exported through a wrapper. You still never add them to your own `package.json`.
::
